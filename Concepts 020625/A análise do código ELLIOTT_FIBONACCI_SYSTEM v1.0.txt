Elliott e HMM-like (Seções _generate_wave_candidates_hmm, _validate_elliott_rules):

Rigor: Window_sizes=[6,8,10] + combos gera candidatos O(n^2), mas falha em <6 swings (sims: filtered_swings~3-4). Validação retorna (valid, score), weighted_score / total_weight >=0.7 é laplaciano (assertivo). Wave5_divergence <1.618*wave3 alinha a teoria. Confiança: clarity=1-CV(points) + trend_alignment (slope_norm * R² multi-window) + regime_factor.
Validade: Estudos 2024-2025 mostram Elliott + ML (e.g., indicators para crypto) melhora predição medium-term em S&P/BTC, mas baixa em HFT devido a ruído; HMM implícito (sequenciação) é bom, mas use full hmmlearn para states (impulsive/corrective).


Benford Aprimorada (Seção BenfordStatisticalValidator):

Fundamentos: χ² + KL robusto, mas warnings: log(p_obs/p_exp) com p_obs=0 → -inf; fix ε=1e-9. Compliant se p>=0.01 & KL<0.1; sims lognormal: KL~0.01 compliant.
Validade: Pesquisas 2025 confirmam em anomaly detection (valuations vs market, ML hybrid para fraud), mas divergences em firm-level < esperado; integra unsupervised para +sensibilidade.


Regime e Bayes (Seções MarketRegimeDetector, BayesianConfidenceEngine):

Regime: Multi-window slope * R² médio é bom; Hurst para efficiency (0.5 random walk) detecta ineficiência (threshold0.7). Erro: linregress em small y (fix clip). CRASH: vol>3base & strength<-0.8 laplaciano.
Bayes: Beta(1,1) + Laplace correto; credible ppf(95%) HPD. Unused em validation, mas potencial para pattern_accuracy.


Fib e Risco (Seções FibonacciRatioCalculator, DynamicRiskEngine):

Clusters threshold=0.005 intervalar; Kelly half (0.25f) mitiga (literatura: reduz drawdown 50% em algo). Adjusted_win=0.65*conf conservador; risk_score ponderado (base 0.3 + pattern 0.25 etc.) é hamiltoniano (min risco-retorno path). Decimal prec=100 overhead ~20x, mas estabilidade (cond<10^12).

# Fix linregress em MarketRegimeDetector._calculate_advanced_trend_strength
def _calculate_advanced_trend_strength(self, prices: List[float]) -> float:
    if len(prices) < 20:  # Min para smallest window
        return 0.0
    windows = [min(w, len(prices)) for w in [20, 30, 50]]
    strengths = []
    for window in windows:
        x = np.arange(window)
        y = prices[-window:]
        if len(y) != len(x):  # Explicit check
            continue
        slope, _, r_value, _, _ = stats.linregress(x, y)
        normalized_slope = slope / np.mean(y) if np.mean(y) != 0 else 0
        strengths.append(normalized_slope * (r_value ** 2))
    return float(np.mean(strengths)) if strengths else 0.0

# Fix KL div0 em Benford._calculate_kl_divergence
def _calculate_kl_divergence(self, digit_count: Counter, total_observations: int) -> float:
    epsilon = 1e-9
    kl_div = 0.0
    for digit in range(1, 10):
        p_obs = (digit_count.get(digit, 0) + epsilon) / (total_observations + 9*epsilon)
        p_exp = self.expected_distribution[digit]
        kl_div += p_obs * np.log(p_obs / p_exp)
    return kl_div

# Adicione em _select_best_wave: validate rules
def _select_best_wave(self, candidates: List[ElliottWave], regime: MarketRegime) -> Optional[ElliottWave]:
    validated = []
    for c in candidates:
        valid, score = self._validate_elliott_rules(c)
        c.confidence *= score
        if valid:
            validated.append(c)
    return max(validated, key=lambda w: w.confidence) if validated else None
